# 中短篇新对话输入区布局设计

## 问题

右侧对话面板使用固定四行 Grid，但历史对话列表是可选节点。列表收起后，消息区和输入区依次占用了错误的网格行，导致输入区进入可伸缩行，模式按钮、文本框和发送按钮被纵向拉伸。

## 设计

右侧对话面板改为纵向 Flex 容器。标题栏、历史列表和输入区使用内容高度，消息记录区使用 `flex: 1`、`min-height: 0` 并独立滚动。这样无论历史列表是否显示，输入区都不会改变职责或尺寸。

输入区继续使用内部 Grid 组织引用、模式、文本框和操作行，但不占用剩余高度。模式按钮保持横向紧凑排列，发送按钮放入右对齐操作行并保持普通按钮高度。外层右栏仍与工作区等高。

## 验收

- 新对话与已有消息对话的输入区均保持紧凑。
- 历史列表显示或隐藏时，消息区只占用剩余空间。
- 文本框保持五行自然高度，发送按钮不铺满宽度或高度。
- 消息区可以独立滚动，右栏整体不产生额外页面滚动。
