# 中短篇对话会话生命周期修复 Implementation Plan

**目标：** 消除空历史会话，并让历史切换具有明确、不会串话的加载反馈。

**实现方式：** 用纯函数筛选可见会话；新增本地新对话重置动作；首次发送时复用现有创建会话流程；历史切换使用目标 ID 防止过期请求更新界面。

## 任务 1：建立回归测试

- 在 `short-story-chat-pane-source.test.ts` 中断言新对话按钮使用本地重置动作，而不是直接创建会话。
- 为可见历史筛选函数新增单元测试，覆盖空会话过滤和非空会话顺序保留。
- 运行目标测试并确认因缺少实现而失败。

## 任务 2：实现会话生命周期

- 新增可见会话筛选纯函数并接入列表加载。
- 新增本地新对话动作，清除当前会话和旧消息。
- 保留首次发送时创建服务端会话的行为。
- 历史切换时立即关闭列表、清空旧消息，并限制过期请求更新状态。
- 运行目标测试并确认通过。

## 任务 3：真实页面和完整验证

- 在指定中短篇工作区验证新对话不产生空历史、空会话不显示、历史切换有明确加载反馈。
- 运行 `npm run test:web`、`npm run typecheck`、`npm run lint`、`npm run build` 和 `git diff --check`。
- 仅提交本次相关文件，不纳入用户已有的其他未提交改动。
