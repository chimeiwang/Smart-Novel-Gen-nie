# 创作资料库需求

## 目标

为长篇中文小说提供可持续维护的创作资料库，让正文写作、Agent 上下文、校验和文风生成都有稳定数据来源。

## 工作台资料聚合

工作台进入小说后，需要聚合以下数据：

- 章节和章节进展；
- 章节质量检查；
- 角色、角色关系、角色经历；
- 物品、地点、势力、术语；
- 故事背景；
- 世界设定；
- 作品圣经；
- 文本总纲；
- 结构化大纲节点；
- 剧情进度；
- 参考资料；
- 已应用文风。

这些数据既服务前端展示，也服务 Agent 上下文聚合。

## 设定管理

### 角色

用户可以创建、编辑、删除角色。

角色字段：

- 基础信息：姓名、别名、性别、年龄、外貌、性格、身份、背景。
- 角色不变量：核心欲望、行为边界、说话习惯、关键关系原则、当前短期目标。
- 实力相关：实力等级、战斗能力、特殊技能。
- 当前状态：active、missing、dead、imprisoned、unknown。
- 状态备注。
- 所属势力。

角色不变量用于 OOC 校验，不应被近期剧情轻易覆盖。

### 角色关系

用户可以维护角色之间的关系。

关系类型：

- 家族；
- 师徒；
- 朋友；
- 敌对；
- 盟友；
- 恋人/伴侣；
- 竞争对手；
- 上下级；
- 熟人；
- 其他。

关系字段：

- 起点角色；
- 目标角色；
- 关系类型；
- 亲密度；
- 描述；
- 开始时间/章节；
- 结束时间/章节。

业务规则：

- 关系不能跨小说创建。
- 目标角色必须存在。

### 角色经历

用户可以维护角色经历。

字段：

- 角色；
- 可选章节 ID；
- 经历内容；
- 排序。

用途：

- 记录角色在章节中的事实变化；
- 为 Agent 提供阶段性状态；
- 避免用最近正文直接覆盖角色核心设定。

### 物品

字段：

- 名称；
- 别名；
- 类型；
- 稀有度；
- 效果；
- 来源；
- 描述；
- 持有人。

### 地点

字段：

- 名称；
- 别名；
- 类型；
- 父级地点；
- 气候；
- 文化；
- 描述。

### 势力

字段：

- 名称；
- 别名；
- 类型；
- 根据地；
- 描述。

### 术语

字段：

- 术语；
- 定义；
- 分类。

## 作品级文本资料

### 故事进展

故事进展保存在 Novel.storyProgress。

业务规则：

- 最多保留 30000 字符。
- 用于记录全书当前进展，帮助 AI 判断“现在写到哪里”。

### 章节进展

章节进展保存在 ChapterProgress。

用途：

- 记录当前章节关键事件；
- 记录本章伏笔、转折、已完成内容；
- 辅助续写和检查。

### 故事背景

故事背景保存在 StoryBackground。

用途：

- 记录作品基础背景；
- 服务设定 Agent 和写作 Agent 上下文。

### 世界设定

世界设定保存在 WorldSetting。

用途：

- 记录世界规则、力量体系、社会结构等。

### 作品圣经

作品圣经用于记录商业定位与写作约束。

字段：

- 题材/频道；
- 目标读者画像；
- 核心卖点；
- 读者承诺；
- 爽点/情绪收益模型；
- 雷点/禁忌；
- 对标作品或参考方向；
- 其他备注。

用途：

- 给编辑 Agent 提供商业判断依据；
- 给校验 Agent 提供读者承诺和禁忌边界；
- 给写作 Agent 提供风格和方向约束。

## 大纲管理

系统同时支持文本总纲和结构化大纲。

### 文本总纲

文本总纲保存在 Outline.content。

需求：

- 用户可以直接编辑保存。
- Agent 创建或重构大纲时，也可通过待审核草案更新文本总纲。

### 结构化大纲

结构化大纲保存在 OutlineNode。

节点类型：

- stage：阶段/卷；
- plot_unit：剧情单元；
- chapter_group：章节组。

层级规则：

~~~mermaid
flowchart TD
    A["stage 阶段/卷"] --> B["plot_unit 剧情单元"]
    B --> C["chapter_group 章节组"]
~~~

业务规则：

- stage 必须是顶层节点。
- plot_unit 必须挂在 stage 下。
- chapter_group 必须挂在 plot_unit 下。
- 节点标题不能为空。
- 删除节点前必须先删除或移动子节点。
- 修改节点类型时，已有子节点必须仍满足新类型允许的层级。

节点字段：

- 标题；
- 内容；
- 排序；
- 类型；
- 状态：planned、in_progress、completed、skipped；
- 预估字数；
- 实际字数；
- 可关联章节。

## 剧情进度

剧情进度保存在 PlotProgress。

字段：

- 当前阶段；
- 当前目标；
- 当前冲突；
- 下一个里程碑。

用途：

- 前端展示当前写作方向；
- Agent 判断下一步推进什么；
- 章节规划和正文生成时作为上下文。

## 伏笔管理

数据库支持 Foreshadowing。

字段：

- 名称；
- 埋设位置；
- 埋设内容；
- 预期回收方式；
- 回收位置；
- 状态：active、paid_off、abandoned。

当前主要通过 Agent 草案链路管理伏笔，不是普通手动面板的主入口。

## 参考资料

用户可以新增参考资料。

字段：

- 标题；
- 类型：note、web、book、image、custom；
- 内容；
- 来源 URL。

用途：

- 保存世界观草稿、素材、外部资料摘要；
- 供 Agent 上下文参考。

## 文风画像

### 文风库

用户可以创建文风，并上传 TXT 参考资料。

上传规则：

- 只支持 .txt 文件。
- 文件内容不能为空。
- 单个文件内容最大 50MB。
- 文件保存到本地 uploads/styles 目录。
- 系统记录原始文件名、本地路径和去空白字符数。

### 异步画像生成

用户可以基于文风参考资料生成画像。

流程：

~~~mermaid
flowchart TD
    A["创建文风"] --> B["上传 TXT 参考资料"]
    B --> C["点击生成画像"]
    C --> D["创建 StylePortraitTask"]
    D --> E["异步读取参考资料"]
    E --> F["Portrait Agent 生成五类画像字段"]
    F --> G{"生成结果"}
    G -->|"成功"| H["写入 WritingStyle 画像字段"]
    G -->|"失败"| I["记录 errorMessage"]
    H --> J["前端轮询任务状态并刷新"]
    I --> J
~~~

画像字段：

- 创作方法论；
- 独特标记；
- 生成风格；
- 表达特征；
- 风格特征；
- Markdown 汇总。

### 分节重生成和编辑

用户可以针对单个画像分节重新生成，也可以手动编辑画像分节。

### 应用文风

用户可以把一个 WritingStyle 应用到小说。

业务规则：

- 小说记录 appliedStyleId。
- 工作台展示已应用文风。
- 写作上下文可读取已应用文风。

## 手动更新与 Agent 更新边界

~~~mermaid
flowchart LR
    A["用户在面板手动编辑"] --> B["Server Action 直接保存正式数据"]
    C["Agent 生成设定/大纲/伏笔变更"] --> D["ReviewArtifact agent_updates"]
    D --> E["用户审核"]
    E -->|"应用"| F["executeUpdates 写入正式数据"]
    E -->|"丢弃"| G["删除草案"]
~~~

规则：

- 手动面板编辑是用户直接操作，可以直接保存。
- Agent 产物必须进入待审核草案。
- Agent 草案应用前不能混入正式小说上下文，除非通过明确 artifact 读取入口。

## 验收标准

- 用户可以维护角色、关系、经历、物品、地点、势力、术语。
- 角色不变量、实力字段、状态字段可以保存并在刷新后恢复。
- 用户可以维护故事进展、章节进展、故事背景、世界设定、作品圣经。
- 用户可以维护文本总纲和三层结构化大纲。
- 系统拒绝非法结构化大纲层级。
- 用户可以新增参考资料。
- 用户可以创建文风、上传 TXT、生成画像、编辑画像并应用到小说。
- Agent 生成的设定、大纲、伏笔变更不会绕过 ReviewArtifact 直接落库。
