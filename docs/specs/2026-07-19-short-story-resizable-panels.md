# 中短篇三栏宽度调整规格

## 目标

中短篇工作区的写作进度、正文画布和写作对话保持同一行展示。用户可以拖动两条分隔线调整三栏宽度，左右栏具有明确的最小和最大宽度，中央正文区自动占用剩余空间。调整结果保存到当前浏览器，下次打开同一作品时自动恢复。

## 技术决策

- 使用 `react-resizable-panels`，不实现 Pointer Event、鼠标捕获、键盘调整或宽度换算逻辑。
- 使用水平方向的 `Group`，三个 `Panel` 与两个 `Separator` 必须是直接子节点。
- 左栏默认 `280px`、最小 `220px`、最大 `360px`。
- 中栏默认占用剩余空间、最小 `640px`，是唯一保持相对宽度的面板。
- 右栏默认 `400px`、最小 `320px`、最大 `520px`。
- 左右栏使用像素宽度保持策略；窗口尺寸变化时仍服从最小、最大约束。
- 面板不可折叠，避免版本历史或写作对话被完全隐藏。

## 浏览器持久化

- 存储键包含版本和小说 ID：`inkforge:short-story-panel-layout:v1:<novelId>`。
- 只在第三方组件触发 `onLayoutChanged` 后保存最终布局，拖动过程中不反复写入。
- 页面初始化时读取并解析布局；仅接受包含左、中、右三个有限正数的对象。
- 浏览器禁用、拒绝或损坏 `localStorage` 时回退到默认布局，工作区仍可使用。
- 读取到超出约束的旧布局时交由 `react-resizable-panels` 按当前约束修正，不绕过组件自行计算宽度。

## 交互与视觉

- 分隔线视觉为克制的 1px 边界，实际命中区域由组件提供并保持易于拖动。
- hover、拖动中和键盘 focus 时显示明确的墨色高亮，不引入大面积色块。
- 分隔条保留组件生成的 `role="separator"` 和键盘交互能力。
- 三栏继续各自滚动，调整宽度不能重置正文阅读位置或写作对话内容。
- 仅修改中短篇工作区；长篇工作区布局保持不变。

### 写作对话内部布局

- 右侧写作对话面板与工作区保持等高，但内部只有消息记录区可以占用剩余高度。
- 会话标题栏、可选的历史对话列表和底部输入区均按内容保持紧凑高度，不得被拉伸。
- 新对话没有消息时，空状态位于消息记录区；不得因此把模式按钮、输入框或发送按钮拉高。
- “讨论”“修改大纲”“修改正文”保持横向紧凑排列；输入框保持约五行的自然高度。
- 发送按钮保持普通按钮高度并靠右显示，不铺满输入区宽度或高度。

## 验收标准

- 两条分隔线均可拖动，左右栏不会超过各自最大宽度。
- 刷新页面后恢复拖动结束时的布局；不同作品使用不同存储键。
- 损坏或不可访问的浏览器存储不会导致页面崩溃。
- 源码不存在自写的 `pointerdown`、`mousemove`、`mousedown` 或拖动宽度计算。
- 分隔线可以获得焦点并通过键盘调整。
- 相关 Web 测试、TypeScript、ESLint 和生产构建通过。
