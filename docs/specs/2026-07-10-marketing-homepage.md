# 官网首页与注册入口

## 背景

当前根路由 `/` 承载已登录作者的小说列表和新建小说入口。未登录用户需要先访问 `/login` 才能注册或登录，产品能力说明缺少独立官网首页承载。

用户希望网站首页从工作台入口调整为官网首页，并保留清晰注册入口。首页内容必须基于当前 InkForge 已有功能，不写未落地能力。

## 当前项目事实

- InkForge 是面向中文小说作者的本地创作工作台。
- 根路由 `/` 当前查询当前用户小说，并渲染 `NovelListClient`。
- `/login` 使用 `LoginForm`，同一组件支持登录和注册切换。
- 注册成功会创建用户、发放 1000 积分、写入会话 Cookie 并刷新首页。
- 已有能力包括小说/章节管理、创作规划、设定库、文本总纲、三层结构化大纲、剧情进度、文风画像、参考资料、AI 写作会话、智能协作、待审核草案、章节规划、一致性终检、积分和用量记录。
- 前端设计规范要求产品型工作台风格，克制、清楚、信息密度适中，避免大面积深色营销页和夸张渐变。

## 目标

- 将 `/` 调整为面向未登录访客和已登录用户都可访问的官网首页。
- 首页用当前真实功能规划产品说明，突出“从创意到可审核草案再到正式写入”的工作流。
- 官网文案必须使用产品语言，不向用户展示 `GraphState`、`embedding`、`TokenUsage`、`CreativeOperation`、`ReviewArtifact` 等实现名词。
- 官网首页暂不宣传中短篇规划；也不能把产品定位写成只面向长篇写作。
- 提供显著注册入口，未登录用户点击后直接进入注册模式。
- 保留已登录作者访问小说列表、新建小说、文风库、充值等工作台能力的入口。
- 将原 `/` 小说列表迁移到独立工作台首页 `/dashboard`。

## 非目标

- 不新增新的认证体系、注册字段或商业定价页面。
- 不改变小说、章节、智能协作、草案审核、计费或数据库模型。
- 不新增移动端专项适配；仍按当前 PC 优先规则设计。
- 不宣称尚未实现的发布、多人协作、云同步、移动端或收益分析能力。

## 设计方案

1. 路由调整：
   - `/` 渲染官网首页。
   - `/dashboard` 渲染原小说列表工作台。
   - `/login?mode=register` 打开注册模式。
   - 中间件公开放行 `/`，其他工作台路由继续要求登录。
2. 登录跳转：
   - `/login` 已登录时跳转 `/dashboard`。
   - 登录或注册成功后进入 `/dashboard`。
3. 官网首页内容：
   - 首屏：品牌、定位、主要 CTA、产品工作流预览。
   - 工作流：项目规划、资料库/大纲、智能协作、草案审核、质量终检。
   - 功能域：作品规划、结构化大纲、文风画像、参考资料、写作会话、积分与用量。
   - 注册说明：注册即送 1000 积分，仅陈述当前产品事实。
4. 视觉：
   - 复用 `DESIGN.md` 的浅色、hairline 边框、产品面板和紧凑按钮风格。
   - 首页更像产品说明页，不做纯营销大 hero，不用夸张渐变或虚构截图。
   - 顶部导航和首屏内容使用同一个内容宽度，不为浮动用户信息条预留额外右侧偏移；官网页不显示全局浮动用户信息条。

## 验收标准

- 访问 `/` 时显示官网首页，而不是小说列表。
- 首页存在注册入口，链接到 `/login?mode=register`。
- 访问 `/dashboard` 时显示原有小说列表和新建小说入口。
- 已登录用户访问 `/login` 会跳转 `/dashboard`。
- 登录或注册成功后进入 `/dashboard`。
- 文案只描述当前项目已存在能力。
- `npm run typecheck` 和 `npm run lint` 通过。
