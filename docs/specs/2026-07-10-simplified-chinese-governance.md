# 仓库简体中文治理规格

## 背景

仓库已经要求后续对话使用简体中文，但尚未把代码注释、文档、计划、审计和维护备注统一纳入仓库级语言治理。近期新增的 Python 后端重构规格和实施计划仍包含大量英文自然语言叙述，新增 Python 包也存在英文文档字符串和断言消息。这会造成维护语言不一致，并削弱文档的可读性和权威性。

## 当前项目事实

- 根目录 `AGENTS.md` 是仓库级开发护栏，已经要求后续对话使用简体中文。
- 根目录 `DOCS.md` 是文档治理权威，定义文档类型、权威层级、变更流程和验收规则。
- `docs/specs/2026-07-10-python-backend-rewrite.md` 是当前 Python 后端重构设计规格，主体为中文，但自然语言中仍混用英文术语。
- `docs/plans/2026-07-10-python-backend-rewrite.md` 是对应实施计划，包含 20 个任务、执行命令和验收门槛，当前自然语言叙述主要为英文。
- 仓库包含历史文档与归档材料，本次变更不要求一次性翻译所有未被修改的历史文件。

## 目标

- 在仓库最外层权威规则中明确：仓库内所有自然语言代码注释、文档字符串、文档正文、计划、规格、审计、备注、示例说明和用户可见维护说明一律使用简体中文。
- 明确代码标识符、协议字段、命令、文件路径、环境变量、第三方产品或库专名，以及无法改写的外部原文引用不受翻译要求约束。
- 要求所有新增或修改内容不得使用英文或繁体中文进行自然语言叙述。
- 要求历史文件在被修改时同步清理其受影响部分，但不要求一次性翻译全部归档内容。
- 把刚新增的 Python 后端重构规格、实施计划和 Python 包说明改为简体中文，同时保持原有架构、任务顺序、命令和验收语义不变。

## 非目标

- 不翻译代码标识符、测试函数名、协议字段、实际命令、文件路径、环境变量和第三方专名。
- 不修改 Python 后端重构的架构设计、任务范围、任务顺序、测试命令或验收标准。
- 不在本次变更中翻译全部历史归档、第三方许可证或无法改写的外部原文。
- 不建立独立于 `AGENTS.md` 和 `DOCS.md` 的第二套语言规范。

## 设计方案

1. 在根目录 `AGENTS.md` 的最外层重要规则处加入仓库级简体中文铁律，使其约束代码、文档和维护说明。
2. 在 `DOCS.md` 中把相同规则纳入文档新增、文档修改和验收流程，并由本文件继续作为唯一文档治理权威。
3. 翻译 `docs/specs/2026-07-10-python-backend-rewrite.md` 中的英文自然语言术语，保留必要的技术专名、路径、字段和协议值。
4. 翻译 `docs/plans/2026-07-10-python-backend-rewrite.md` 的标题、说明、任务、步骤、预期结果、提交说明和最终门槛，完整保留 20 个任务、全部勾选项、顺序、文件路径、代码块、命令和技术要求。
5. 清理本次新增 Python 文件中的英文自然语言文档字符串、注释和断言消息；代码标识符保持不变。
6. 使用结构计数、英文标题搜索、繁体常见字搜索和 `git diff --check` 验证变更。

## 影响范围

- 根级权威文档：`AGENTS.md`、`DOCS.md`。
- 新增治理规格：`docs/specs/2026-07-10-simplified-chinese-governance.md`。
- 当前重构规格与计划：`docs/specs/2026-07-10-python-backend-rewrite.md`、`docs/plans/2026-07-10-python-backend-rewrite.md`。
- 本次新增的 Python 包文件与仓库结构测试。
- 后续所有代码、文档、计划、规格、审计和维护说明的自然语言内容。

## 验收标准

- `AGENTS.md` 在最外层明确仓库级简体中文铁律，并引用 `DOCS.md` 作为文档治理权威。
- `DOCS.md` 的新增、修改和验收规则明确要求自然语言使用简体中文，并列出允许保留原文的技术例外。
- 两份 Python 后端重构文档不再使用英文或繁体中文进行自然语言叙述。
- 实施计划仍包含 20 个 `### 任务` 标题，勾选项数量不减少，任务顺序、命令、路径和验收要求不丢失。
- 本次新增 Python 文件中的自然语言文档字符串、注释和断言消息使用简体中文。
- 搜索 `Goal`、`Architecture`、`Task`、`Step`、`Files`、`Run`、`Expected`、`Final Completion Gate` 和 `Spec compliant` 等明显英文叙述标题时结果为零。
- 常见繁体中文字符搜索结果为零。
- `git diff --check` 通过。
